# Internacionalização (i18n)

CircleHood Booking suporta três idiomas: **Português Brasileiro (PT-BR)**, **Inglês US (EN-US)** e **Espanhol (ES-ES)**.

## Stack

- **`next-intl` v4** — tradução para Server e Client Components
- **`localePrefix: 'as-needed'`** — PT-BR usa URLs sem prefixo (`/dashboard`), EN-US e ES-ES usam prefixo (`/en-US/dashboard`, `/es-ES/dashboard`)

## Estrutura de arquivos

```
messages/
  pt-BR.json     ← Português (padrão)
  en-US.json     ← Inglês
  es-ES.json     ← Espanhol

src/
  i18n.ts        ← getRequestConfig + locales + defaultLocale
  navigation.ts  ← createNavigation (Link, redirect, useRouter, usePathname)
  middleware.ts  ← chain: Supabase auth + next-intl locale routing
  components/
    locale-switcher.tsx  ← Componente de troca de idioma (sidebar/mobile header)
  app/
    [locale]/
      layout.tsx          ← NextIntlClientProvider por locale
      (auth)/             ← Páginas de login/register
      (dashboard)/        ← Dashboard protegido
      (public)/           ← Páginas públicas dos profissionais
```

## Adicionar uma nova tradução

1. Adicione a chave nos três arquivos `messages/*.json` no mesmo namespace
2. Use `useTranslations('namespace')` em Client Components
3. Use `useTranslations('namespace')` (importado de `next-intl`) em Server Components

### Server Component
```tsx
import { useTranslations } from 'next-intl';

export default async function MyPage() {
  const t = useTranslations('myNamespace');
  return <h1>{t('myKey')}</h1>;
}
```

### Client Component
```tsx
'use client';
import { useTranslations } from 'next-intl';

export function MyComponent() {
  const t = useTranslations('myNamespace');
  return <button>{t('save')}</button>;
}
```

## Troca de idioma

**Via UI:** O componente `LocaleSwitcher` aparece na barra lateral do dashboard e no header mobile. Ao trocar o idioma, o usuário é redirecionado para a mesma página com o novo prefixo de locale.

**Via Configurações:** Na página de Configurações → Conta → "Idioma do Dashboard", o profissional pode salvar o idioma preferido. O valor é salvo em `professionals.locale` no banco de dados e o dashboard redireciona para o novo locale automaticamente.

## Idiomas suportados

| Locale | Moeda | Data | Hora |
|--------|-------|------|------|
| pt-BR  | R$    | DD/MM/AAAA | 24h |
| en-US  | $     | MM/DD/YYYY | 12h AM/PM |
| es-ES  | €     | DD/MM/AAAA | 24h |

## Bot WhatsApp multilíngue

O bot detecta o idioma da mensagem na primeira mensagem do cliente (usando Claude Haiku). Se o cliente escrever em inglês, o bot responde em inglês; em espanhol, em espanhol; em português, em português.

- A detecção só ocorre em mensagens de saudação (`isFirstMessage`)
- Para cold-start (histórico não propagado ainda), a detecção é ignorada para não adicionar latência
- O idioma detectado é salvo em `bookings.client_locale`

## Database

```sql
-- Idioma preferido do profissional
ALTER TABLE professionals ADD COLUMN locale VARCHAR(10) DEFAULT 'pt-BR';

-- Idioma do cliente no agendamento
ALTER TABLE bookings ADD COLUMN client_locale VARCHAR(10) DEFAULT 'pt-BR';
```

## Adicionar novo idioma

1. Criar `messages/xx-XX.json` com todas as chaves (copiar do `pt-BR.json`)
2. Adicionar `'xx-XX'` ao array `locales` em `src/i18n.ts`
3. Adicionar opção em `src/components/locale-switcher.tsx`
4. Adicionar opção em `LOCALE_OPTIONS` em `src/components/dashboard/settings-manager.tsx`
5. Atualizar CHECK constraint na migration do DB
