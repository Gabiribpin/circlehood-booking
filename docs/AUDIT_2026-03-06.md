## CRITICAL

### C-01: 6 tabelas sem RLS habilitado
- **Tabelas**: `notification_queue`, `notification_logs`, `waitlist`, `service_packages`, `loyalty_cards`, `loyalty_transactions`
- **Evidência**: `supabase/migrations/20250217000000_sprint7_automations.sql`
- **Impacto**: Qualquer usuário autenticado pode ler/modificar dados de TODOS os profissionais nestas tabelas
- **Correção**: Migration com `ALTER TABLE x ENABLE ROW LEVEL SECURITY` + policies por professional_id
- **Esforço**: 1h

### C-02: Tabelas admin-only sem deny-all RLS
- **Tabelas**: `sales_leads`, `sales_conversations`, `sales_messages`, `control_center_entries`
- **Evidência**: `supabase/migrations/20260227000004_sales_leads.sql`, `supabase/migrations/20260302000004_control_center_entries.sql`
- **Impacto**: Se código acidentalmente usar client regular (não admin), dados ficam expostos
- **Correção**: `ENABLE RLS` + policy `USING (false)` (service role bypassa)
- **Esforço**: 30min

### C-03: Evolution API instance parameter injection
- **Evidência**: `src/app/api/evolution/get-qrcode/route.ts:14,30`
- **Impacto**: Parâmetro `instance` da query string é interpolado diretamente na URL `${url}/instance/qrcode/${instance}` — path traversal possível
- **Correção**: Validar com regex: `z.string().regex(/^[a-zA-Z0-9_-]+$/)`
- **Esforço**: 15min

### C-04: Cron bearer token vulnerável a timing attack
- **Evidência**: `src/app/api/cron/send-reminders/route.ts:68-73` + 7 outros crons
- **Impacto**: Comparação `authHeader !== \`Bearer ${cronSecret}\`` permite timing attack para adivinhar o secret
- **Correção**: Usar `timingSafeEqual()` de `crypto`
- **Esforço**: 30min

## HIGH

### H-01: WhatsApp tokens armazenados em plaintext
- **Evidência**: `supabase/migrations/20260218000002_whatsapp_system.sql:6-7` — `access_token TEXT`, `verify_token TEXT`
- **Também**: `evolution_api_key TEXT` em `20260218000005_evolution_api_support.sql:5`
- **Impacto**: Acesso ao DB expõe tokens da Evolution API
- **Correção**: Usar pgcrypto ou encryption at app level (já existe `token-encryption.ts`)
- **Esforço**: 2h

### H-02: Sem rate limiting no GET /api/available-slots
- **Evidência**: `src/app/api/available-slots/route.ts:1-132`
- **Impacto**: Enumeração de todos os slots de todos os profissionais
- **Correção**: Rate limit por IP: 100 req/min
- **Esforço**: 15min

### H-03: POST /api/testimonials permite submissão sem auth
- **Evidência**: `src/app/api/testimonials/route.ts:87-100`
- **Impacto**: Spam de testemunhos para qualquer profissional
- **Correção**: Rate limit por IP + validação de email
- **Esforço**: 30min

### H-04: Sem rate limiting no POST /api/auth/verify-email
- **Evidência**: `src/app/api/auth/verify-email/route.ts:1-50`
- **Impacto**: Brute-force de tokens de verificação
- **Correção**: Rate limit por IP: 10 req/h
- **Esforço**: 15min

### H-05: Sem rate limiting no envio de email campaigns
- **Evidência**: `src/app/api/email-campaigns/[id]/send/route.ts:1-167`
- **Impacto**: Usuário autenticado pode triggerar blast massivo de emails
- **Correção**: Rate limit por professional_id: 5/h
- **Esforço**: 15min

### H-06: Revolut webhook usa client autenticado em vez de admin
- **Evidência**: `src/app/api/webhooks/revolut/route.ts:7`
- **Impacto**: Webhook não tem permissão para atualizar tabela professionals (RLS bloqueia)
- **Correção**: Trocar `createClient()` por `createAdminClient()`
- **Esforço**: 5min

### H-07: Detecção de refund parcial usa boolean em vez de amount
- **Evidência**: `src/app/api/webhooks/stripe-deposit/route.ts:87`
- **Impacto**: `charge.refunded` é true se QUALQUER refund existe, não se é total. Refund parcial marcado como total.
- **Correção**: `const isFullRefund = charge.amount_refunded === charge.amount;`
- **Esforço**: 5min

### H-08: Input validation ausente em múltiplos endpoints
- **Endpoints afetados**:
  - `POST /api/reschedule/[token]/change` — new_time sem validação de formato (`route.ts:11,45`)
  - `POST /api/waitlist` — notes e preferred_dates sem max length (`route.ts:80-90`)
  - `PUT /api/testimonials` — updateData aceita campos arbitrários (`route.ts:218`)
  - `DELETE /api/testimonials` — ID sem validação UUID (`route.ts:277`)
  - `POST /api/gallery/upload` — title/description sem max length (`route.ts:73-75`)
- **Correção**: Adicionar schemas Zod em cada endpoint
- **Esforço**: 1.5h

### H-09: CSV injection no import de contatos
- **Evidência**: `src/app/api/contacts/import/route.ts:40-54`
- **Impacto**: Fórmulas Excel (=CMD) no CSV podem executar quando exportado
- **Correção**: Sanitizar: prefixar `'` em valores que começam com `=`, `+`, `-`, `@`
- **Esforço**: 30min

### H-10: Missing error.tsx em 3 route groups
- **Evidência**: Ausente em `(auth)/`, `(admin)/`, `(public)/`
- **Impacto**: Erros de render mostram tela branca ou stack trace
- **Correção**: Criar error.tsx em cada grupo
- **Esforço**: 30min
