Config

Isso puxa Issues do GitHub, escolhe automaticamente a "próxima" (BLOCKER > CRITICAL > HIGH > MEDIUM > LOW > ENHANCEMENT), e mantém um "checkpoint" (onde você parou) sem você pensar.

Segurança: para usar direto no browser, crie um Fine-grained PAT com acesso apenas ao repositório e permissões mínimas: Issues: Read/Write (e opcionalmente Contents: Read se você quiser ler arquivos). Para produção, o ideal é um proxy/GitHub App (token nunca no front).

Regras (fixas)

1) Uma issue por vez.
2) Nunca refatorar por "vontade" antes de zerar BLOCKER.
3) Ideia nova = vira issue (enhancement) e volta para a atual.
4) "Done" só com comentário técnico + fechamento.

Agora

Conecte para a roda escolher automaticamente a próxima issue.
